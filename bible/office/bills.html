Image formats

Each scanned bill or statement should be saved in two files, one at high res, one at low res, as follows:

High Res (filenameb.gif), is a 256 colour gif file, at 144dpi. It is produced as follows:

1. scan original at 144dpi full colour scan (RGB)
2. convert to indexed colour using Selective, 256 colours, Diffusion Dithering
3. save as gif file (non interlaced)

Low Res (filename.gif), is an 8 colour, greyscale (no colour), gif file at 72dpi:

1. open original scan (144dpi, full colour RGB)
2. convert to greyscale
3. convert to indexed colour, 8 colours, selective, dithering
4. save as gif file (non interlaced)

Remember that whenever you resize an image to make sure you've put the colour mode into either "RGB" or "Greyscale" before you resample the image.

Scanned Bill Filename Convention:
company[-account]_YYYYMMDD.ext

where company is the name of the company,
-account is an identifier for the account if we have many with this company, eg vodafone we have -jdr and -apc
-date is the date of issue of the bill in reverse format, year month day
ext is the filename extension, eg .gif

Scanned Bank Statement Filename Convention:
banksa_pagenum_date.gif

note that the date should be the the last day of the statement period. Eg for statement 206, which covers 26.8.00 to 1.9.00 the filename would be:
banksa_p206_20000901.gif (and the b (big) version too)

VA Universal Document ID Numbers:

Before scanning images make sure to write the file name, including the Document ID Number, on the image to be scanned. The document number will take to form of:
vadocXXXXXX
where the X¹s are numbers. For example the fifty-first document to be scanned will be: vadoc000051

When naming a document it is not necessary to put in all the 0¹s.
For example:
chubb_20000921_51.03b.gif
where the vadoc number is vadoc000051 and the page number is 3.

So, when scanning page one of the above document you would write chubb_20000921_51.01b on the first page, chubb_20000921_51.02b on the second page and so on.
