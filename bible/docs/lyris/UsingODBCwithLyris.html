Using ODBC with Lyris

The safest, easiest way to read the Lyris database files is to use ODBC. ODBC is a set of database drivers which allows ODBC capable programs (such as Microsoft Excel, Access, Paradox, Crystal Reports) to view and change data in many different database formats. By creating an ODBC configuration for Lyris, you will be able to read the Lyris databases from any ODBC capable program.

Directions for configuring ODBC for Lyris

ODBC drivers are available for Windows, Mac and Unix. These directions apply to Windows ODBC drivers. ODBC drivers are not included with Windows, but many programs install them automatically. For instance, if you have Microsoft Office, you already have the necessary ODBC driver (the "FoxPro" or "Visual FoxPro" driver).

If you do not have the Visual FoxPro driver, you can obtain it from Microsoft at no charge. Check www.microsoft.com, or download it directly from http://support.microsoft.com/download/support/mslfiles/VFPODBC5.EXE

* The first step is to go to the control panel, and click on the ODBC icon. It may say "ODBC 32" -- that's what you want.

* You will be presented with a dialog box labeled "Data Sources". Click the "Add" button.

* Choose "Microsoft Visual FoxPro Driver" and click OK.

* A window labeled "ODBC Visual FoxPro Setup" will appear. In the Data Source field, enter "Lyris". In the Description field, enter "Lyris". Choose "Free Table Directory" as the Database Type. For Path, choose the "db" subdirectory where you installed Lyris. On most computers, this is "c:\lyris\db". Click Ok.

* You will be back at the "Data Sources" dialog box. You are done, so click the "close" button.

Directions for using the Lyris ODBC driver

These directions apply to Microsoft Access 95. The directions are very similar for Access 97, and for other database programs.

* First, create a blank Microsoft Access Database.

* Next, choose the File/Get External Data/Link Tables menu choice.

* Where the "Link" dialog box lists "Files of Type", choose "ODBC Databases"

* Choose the "Lyris" ODBC definition you just recently created (using the directions above).

* Choose the table you want to connect to. For example, choose "messages."

* When Access asks you for the "Unique Record Identifier" enter the Key field for that table. For the "messages" table, this is "messageid". The Key field for each table is documented in the Lyris Programmer's Guide, in the "Lyris Databases" section, available online at http://www.lyris.com/guide/ The Key field for each table is the field labeled as "PrimaryKey = yes" for each table. For example, for the Messages table, you will see "Messages.MessageID.PrimaryKey = yes". This means that for the "Messages" table, the "MessageID" field is the "Primary Key".

* You are now done. You can click on the Lyris table in Microsoft Access and edit or view it as you wish. If you make any significant changes, we highly recommend that you back up the tables beforehand.
