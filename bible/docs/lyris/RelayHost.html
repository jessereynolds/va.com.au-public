Relay Host

Relay all outbound mail through this mail host rather than doing regular direct delivery.

Some environments need outbound mail to be relayed through a specific host. For instance, your organization might employ a TCP/IP firewall that prevents Lyris from directly delivering mail. In such a case, each application behind the firewall must use a Relay Host to communicate with the outside.

A mail server must be listening on the port of the TCP/IP address that you specify. For this setting, you specify an address/port combination.

You can specify as many relay hosts as you like. Each entry must be separated by a carriage return. If you specify multiple relay hosts, Lyris will automatically distribute the mail relay load between them. If you need to send a great deal of mail through a relay host setup, we recommend that you specify several relay hosts.

Example:

    207.105.6.2 25
