Add

The add command lets a list administrator add a member to their list. The syntax of the add command is:

    add listname emailaddress [user name] [quiet]

The user name and quiet modifiers are optional. Here is an example:

    add jazztalk bob@somewhere.com Bob Smith

If you do not specify a user name, the user will be created with no name in their member record.

If you specify the quiet modifier, no confirming email messages will be sent out. However, if the add command does not succeed, such as if the email address is already a member, you (the person submitting the add command) will be notified. By default, when you add a member with the add command, both the admin and the user receives notification of the action. If a "hello" document exists for the list, that hello document is sent to the user instead of a machine-generated notification. With the quiet modifier, no notification is sent to the user or to the administrator.

The add command works in all instances, even with closed and private lists. A confirmation is not sent out, even if the list is set to ask for confirmations.

The add command is only available to administrators. You must issue a "login" command before using the add command. For more information about login, see Login.
