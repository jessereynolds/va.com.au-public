Rewrite Purge-Id Tag

In Lyris version 2.5x, a footer was by default appended to list postings which included the recipient's email address in square brackets, like so: [email@somewhere.com]. The instructions for unsubscribing in that version was to forward a posting to the unsubscribe address, where Lyris would read this [email@somewhere.com] and unsubscribe the correct person automatically.

As of v3.0, this method of unsubscribing is obsolete, but still provided for backwards compatibility. In v3.0, each recipient has their own private unsubscribe address that looks like this: unsubscribe-jazztalk-34234P@lyris.net.

With version 2.5x, if someone replied to a message on a list, and included their [email@somewhere.com] tag, Lyris needed to remove the square brackets around the email address. If Lyris did not, then a message would be posted to the list with the original recipient's [email@somewhere.com] in it, followed by the new recipient's [email@somewhere.com] address. Then, when people would try to unsubscribe, they might inadvertently unsubscribe the wrong person.

For this reason, Lyris looks for [email@somewhere.com] in the body of messages and automatically removes the square brackets. As of v3.0 this behavior is no longer needed, but people who created their lists using version 2.5x often still have the old unsubscribe directions, and so Lyris needs to continue this behavior.

If you are only using the new unsubscribe mechanism, and not including the [email@somewhere.com] footer in the bottom of list postings, then you can safely disable the "Rewrite Purge-Id Tag" so that Lyris does not rewrite the body of messages where [email@somewhere.com] is detected.

Note: in rare cases a message is posted with an email address in square brackets that was not created with Lyris, and in that case, Lyris still removes the square brackets. This can be confusing to the list members, and so this option can disable the Lyris behavior.
