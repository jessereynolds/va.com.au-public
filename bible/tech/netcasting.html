Ok - to set up for the broadcast you need to:

1/ Open Sorenson - or whatever broadcasting package you are using. Enter the network settings. In our case we set to "Unicast" with the receiver's IP address being that of Llama, our streaming server. Unless you have been given specific ports to use you can just leave the port settings as they are, ie; default. Set buffer delay to 8 seconds which is the minimum that will work.

Set all your other options - ie; source input, compression type, publishing details, etc. In the file menu select "Save As" and save these settings. By saving the settings we can open them again at any time and begin broadcasting straight away.

2/ "Announce" the broadcast by clicking on the "Announce" button. You will be given the choice of either an "annoucement Movie", (a ".mov" file), or an "SDP Text Announcement", (a ".sdp" file). Choose to save as a .sdp file.

3/ FTP the ".sdp" file to the streaming directory on the streaming server - in our case it is the "/movies" directory on Llama.

4/ Now we check to see that it is all working - for this we will need a registered version of QuickTime. Click "Broadcast" on Sorenson and then open QuickTime. Go to "Open URL" in the "File" menu and enter the URL for the ".sdp" file we just uploaded, but leave the ".sdp" suffix off the end. If everything is working QuickTime should open connect to the streaming server and you should be able to hear/see the broadcast.

5/ Now, still in QuickTime and while the broadcast is playing, save the broadcast as a ".mov" file.

6/ FTP the ".mov" file to your server - or whatever server is going to host the html page that the broadcast will be on.

7/ Create an html page for the broadcast. This can be anything but you will need to embed the ".mov" file in it with the "embed" tag. Use code something like the following:

< embed src="my_broadcast.mov" autostart="true" height="16" width="200" />

Of course if the ".mov" file and this html page are not in the same server directory you will need to include the file path to the ".mov" file. The 'autostart="true"' will make the broadcast start as soon as the page is opened.

8/ FTP the html page to the appropriate place.

That should be about it. Note that if your are using more than one microphone you will need to have an audio mixing desk to make sure that all the levels are correct and be able to adjust them individually. Likewise if you are using more than one camera you will need a vision mixer.
